# Les tests

## Contexte

Un dev sans tests n'est pas un dev. en partant de ce principe, il est essentiel de creer des tests!

## Décision

- Pour la librairie de composant, des test de composants seront a créé dans le projet de la librairie avec playwright
- Pour le fonctionnement globale de l'application des tests e2e seront a créé dans un projet dédié avec playwright sous forme de scénario cucumber
- Des tests spécifique pour le/les serveurs pourrais eventuellement etre dévellopé avec une autre libraire

Des rapports seront a généré afin de mettre en évidence, les tests effectué, la couverture de code et les mauvaises pratiques détectés
