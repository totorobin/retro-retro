# Stratégie CSS et Design System

## Contexte

L'utilisation de bibliothèques de composants externes (Vuetify, PrimeVue, etc.) peut poser des problèmes de dépendances à long terme (mises à jour complexes, obsolescence).

## Décision

Aucune bibliothèque de composants ou framework CSS externe (ex: Tailwind, Bootstrap) ne sera utilisé.
Le style sera géré via du CSS natif avec des variables CSS (Custom Properties) pour la gestion des thèmes (couleurs, bordures, polices).
Les variables seront centralisées dans un fichier CSS global.

## Conséquences

- Contrôle total sur l'apparence et le poids de l'application.
- Indépendance vis-à-vis des cycles de mise à jour des bibliothèques tierces.
- Nécessite de recréer les composants de base (boutons, modales, etc.).
